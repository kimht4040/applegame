# 프로젝트: 숫자 사과 줍기 (Find 10 Apples)

정해진 시간 내에 사과에 적힌 숫자들의 합이 10이 되도록 드래그하여 점수를 얻는 안드로이드 퍼즐 게임입니다. 안드로이드의 기본 위젯을 사용하지 않고 Custom View를 직접 구현하여 렌더링 및 터치 이벤트를 처리하는 능력을 배양하는 것을 목표로 했습니다. 또한 게임 데이터와 UI를 분리하여 설계함으로써 코드의 유지보수성과 확장성을 고려했습니다.

## 1. 주요 기능 및 구현 내용

### Custom View 기반의 게임 화면 구현
* **직접 렌더링**: View 클래스를 상속받아 onDraw 메소드를 오버라이드하여 게임 보드, 사과 비트맵, 숫자 텍스트, 사용자의 드래그 영역 등 모든 그래픽 요소를 Canvas에 직접 렌더링했습니다.
* **동적 레이아웃**: onSizeChanged를 통해 디바이스의 화면 크기가 변경되어도 게임 그리드의 비율이 유지되도록 셀 크기를 동적으로 계산했습니다.

### 사용자 인터랙션 및 터치 이벤트 처리 
* **드래그 시스템**: onTouchEvent의 ACTION_DOWN, ACTION_MOVE, ACTION_UP 이벤트를 감지하여 사용자의 드래그 동작을 구현했습니다.
* **영역 시각화**: 사용자가 드래그하는 시작점과 끝점에 따라 실시간으로 선택 영역(사각형)을 계산하고 이를 화면에 표시하여 직관적인 UI를 제공했습니다.

### 게임 로직 및 상태 관리 
* **핵심 알고리즘**: 선택된 사과들의 숫자 합이 정확히 10이 될 경우 해당 사과들을 제거하고 점수를 획득하는 로직을 구현했습니다.
* **제한 시간 설정**: CountDownTimer를 사용하여 2분의 제한 시간을 설정하고 남은 시간을 ProgressBar에 시각적으로 표시했습니다.
* **게임 종료 처리**: 시간이 모두 소진되면 AlertDialog를 통해 게임 종료 팝업을 띄우고 사용자에게 다시 시작 또는 종료 옵션을 제공합니다.

## 2. 기술 스택 
* **언어**: Kotlin 
* **플랫폼**: Android
* **주요 기술**: Custom View, Canvas API, Paint, Bitmap, MotionEvent, Handler, CountDownTimer, AlertDialog

## 3. 설계 및 아키텍처
* **데이터와 UI의 분리**: 게임 그리드의 생성, 숫자 배치, 셀 제거 등 데이터 관련 로직을 GameGrid 클래스로 분리했습니다.
* **클래스 책임 분할**: InGameView는 GameGrid로부터 데이터를 받아와 화면에 그리는 역할에만 집중하도록 설계하여 유지보수성을 높였습니다.

## 4. 성과 및 배운 점
* **좌표계 변환 이해**: 화면 좌표(px)를 게임 로직에서 사용하는 배열 인덱스(row, col)로 변환하는 과정을 통해 물리적 좌표와 논리적 좌표계의 관계를 깊이 이해했습니다.
* **렌더링 파이프라인 학습**: Canvas를 직접 다루면서 안드로이드의 렌더링 파이프라인과 하드웨어 가속의 원리를 학습할 수 있었습니다.
* **객체지향 설계의 중요성**: 클래스의 역할을 명확히 나누는 것이 코드 가독성과 재사용성을 향상시킨다는 점을 체감했으며, 이는 복잡한 시스템 설계의 밑거름이 되었습니다.
* **사용자 경험 고려**: generateEvenlyDistributedRandomGrid 함수를 통해 숫자가 균등하게 분포되도록 설계하여 게임의 난이도를 조절하고 더 나은 사용자 경험을 제공했습니다.

## 5. 향후 개선 사항
* 현재 구현된 dropNumbers()와 refillGrid() 함수를 Handler를 이용한 게임 루프와 연동하여 사과가 아래로 떨어지고 빈칸이 채워지는 기능을 활성화할 계획입니다.
